# Malltopia Game Design Document

Version: 0.1  
Status: MVP planning  
Target: Android, iOS  

## 1. Concept

Malltopia는 여러 테마 안의 스테이지를 차례로 클리어하며 매장을 키우는 방치형 경영 시뮬레이션이다.

테마는 슈퍼마켓, 백화점, 놀이동산처럼 큰 콘셉트 단위로 구성된다. 각 테마 안에는 8개 스테이지를 목표로 구성한다. MVP 검증은 하나의 테마 안에서 3개 스테이지를 먼저 구현하고, 루프와 밸런스가 검증되면 4~8스테이지를 추가한다.

메인 캐릭터는 플레이어의 이동 조작 대상이 아니라, 손님 주문을 받고 생산 위치로 이동하고 상품을 생산한 뒤 손님에게 전달하는 자동 에이전트다. 플레이어는 해금 가능한 칸, 매대, 생산 테이블, 하단 메뉴의 업그레이드 리스트, 다음 스테이지 이동 버튼을 클릭해 운영 루프를 열고 강화한다.

메인 캐릭터는 장비 아이템을 착용해 이동속도, 작업속도, 판매 수익, S급 상품 등장 확률, 직원 능력치 보너스 같은 성장 효과를 얻는다. 장비는 Normal, Rare, Unique, Epic, Legendary, Mythic 6단계 등급으로 구성한다.

## 2. Design Direction

MVP는 자동 에이전트 행동과 유저의 해금/업그레이드 판단을 기본으로 한다.

- 손님은 자동으로 입장한다.
- 메인 캐릭터는 손님에게 자동으로 이동해 주문을 받는다.
- 손님 머리 위 말풍선에는 주문 상품 아이콘이 표시된다.
- 주문 상품을 열 수 있는 해금칸이 깜박이며, 플레이어는 해당 칸을 클릭해 정보를 확인한다.
- 상품 매대와 생산 테이블이 해금되면 메인 캐릭터가 자동으로 생산 위치로 이동한다.
- 생산 중에는 캐릭터 머리 위 원형 진행 게이지가 차오른다.
- 생산이 완료되면 메인 캐릭터가 상품을 들고 손님에게 자동 전달하고 Gold를 획득한다.
- 각 스테이지 시작 시 손님은 1명, 작업자는 메인 캐릭터 1명만 존재한다.
- MVP 기본 흐름은 손님 1명이 입장하고, 상품을 받은 뒤 퇴장하면 다음 손님이 입장하는 순환 구조다.
- 직원 수와 손님 수는 하단 메뉴의 업그레이드 리스트에서 `직원 +1`, `손님 +1` 업그레이드를 구매해 늘린다.
- 직원은 추가된 뒤 메인 캐릭터와 같은 방식의 자동 에이전트 루프를 수행한다.
- 플레이어는 장비, 메인 업그레이드, 스테이지 업그레이드를 통해 메인 캐릭터 능력치와 현재 스테이지 직원 능력치를 함께 성장시킨다.

초반 탭 부스트는 MVP에 포함하지 않는다. 플레이어 입력은 캐릭터 이동 조작이 아니라 해금, 업그레이드, 확장 선택으로 표현한다.

### MVP Implementation Decisions

다음 결정은 MVP 코드 구현 기준으로 고정한다.

| Topic | Decision |
|---|---|
| Player input | 해금칸, 매대, 생산 테이블, 업그레이드 클릭 |
| Main character role | 주문 접수, 생산, 상품 전달을 수행하는 자동 에이전트 |
| Product unlock | 주문 발생 후 해금 가능한 칸이 깜박이고, 클릭 시 정보창 표시 |
| Stand unlock | 상품 해금 후 레벨 1 매대가 열림 |
| Production activation | 매대 옆 생산 테이블 활성화칸을 클릭해야 생산 가능 |
| Production action | 에이전트가 생산 위치로 이동하고 원형 진행 게이지로 생산 시간 표시 |
| Delivery action | 생산 완료 후 에이전트가 상품을 손님에게 전달하고 Gold 획득 |
| Footer menu | 하단 메뉴 아이콘 클릭 시 스테이지 업그레이드 리스트 팝업 표시 |
| Stage upgrades | 생산속도 증가, 보상 증가, 손님 +1, 직원 +1 |
| Stage progression | 현재 스테이지의 모든 상품 매대 최대 레벨 달성 후 다음 스테이지 버튼 점멸, Gold 지불 후 이동 |
| Staff role | 기본은 범용 자동 에이전트. 맵 구조가 카운터/생산 구역으로 나뉘면 서비스/생산 역할 분리 가능 |
| Customer movement | MVP는 waypoint 기반 자동 이동 |
| Product choice | 첫 손님은 첫 상품을 주문해 해금을 유도하고, 이후 손님은 해금되고 생산 가능한 매대 상품 중 균등 랜덤으로 주문 |
| Stage start | 손님 1명, 메인 캐릭터 1명, 직원 0명으로 시작 |
| Stage clear | 현재 스테이지의 모든 상품 매대를 최대 레벨까지 업그레이드하면 다음 스테이지 이동 버튼 점멸 |
| Next stage | 다음 스테이지 이동에는 Gold 비용 필요 |
| Equipment | 장기 핵심 성장축으로 정의하되 MVP 구현 범위는 별도 결정 |
| Save method | 로컬 JSON 저장 |
| Firebase | 버전 관리와 운영 설정용 Remote Config 사용. MVP 유저 진행 데이터는 로컬 저장 |

## 3. Genre

Idle Tycoon / Agent Management Simulation

핵심 경험은 "처음에는 메인 캐릭터 혼자 주문을 처리하던 작은 스테이지가 업그레이드, 손님 증가, 직원 증가, 매대 확장으로 점점 바쁘고 효율적인 공간으로 변하는 만족감"이다.

## 4. Core Fantasy

플레이어는 각 테마의 가장 작은 첫 스테이지에서 시작한다. 처음에는 메인 캐릭터 혼자 주문 접수, 생산, 전달을 처리하지만, 수익을 다시 투자하면서 더 많은 상품 매대, 더 빠른 직원, 더 좋은 장비, 더 큰 테마 공간을 갖춘 쇼핑 제국을 만든다.

## 5. Core Loop

1. 손님 입장
2. 메인 캐릭터가 손님에게 이동
3. 손님 주문 접수
4. 주문 상품 말풍선 표시
5. 해금 가능한 상품칸 또는 생산 테이블 클릭
6. 메인 캐릭터가 생산 위치로 이동
7. 원형 진행 게이지가 차며 상품 생산
8. 생산 완료 상품을 손님에게 전달
9. Gold 획득
10. 하단 메뉴에서 스테이지 업그레이드 구매
11. 상품 매대 레벨 업그레이드
12. 특정 레벨 달성 시 별 획득과 수익 배율 증가
13. 별 조건을 만족한 생산 슬롯 박스 등장
14. 생산 슬롯 박스 터치로 무료 활성화
15. 모든 상품 매대 최대 레벨 달성
16. 다음 스테이지 버튼 점멸
17. Gold를 지불하고 다음 스테이지 이동

## 6. Theme And Stage Progression

게임 진행은 `Theme -> Stage -> Product Stand` 구조를 따른다.

| Layer | Description |
|---|---|
| Theme | 슈퍼마켓, 백화점, 놀이동산 같은 큰 콘셉트 |
| Stage | 한 테마 안의 개별 진행 구간 |
| Product Stand | 스테이지 안에서 판매하는 상품 매대 |

테마 예시:

| Theme | Example Stages |
|---|---|
| Supermarket | 과자 코너, 음료 코너, 신선식품 코너 |
| Department Store | 의류, 화장품, 명품 |
| Amusement Park | 솜사탕, 구슬아이스크림, 닭꼬치, 기념품 |

놀이동산 테마 예시:

| Stage | Starting Stand | Additional Stands | Product Max Level | Next Stage Condition |
|---:|---|---|---:|---|
| 1 | 솜사탕 | 없음 | 25 | 모든 매대 최대 레벨 + 이동 비용 |
| 2 | 구슬아이스크림 | 닭꼬치 | 75 | 모든 매대 최대 레벨 + 이동 비용 |
| 3 | 팝콘 | 핫도그, 츄러스 | 75 | 모든 매대 최대 레벨 + 이동 비용 |
| 4~7 | TBD | TBD | TBD | 모든 매대 최대 레벨 + 이동 비용 |
| 8 | TBD | 최대 10개 가능 | 350 | 테마 클리어 |

각 스테이지는 시작 Gold 5, 손님 1명, 메인 캐릭터 1명으로 시작한다. 첫 매대 해금 비용은 기본 5 Gold로 두어, 유저가 시작 Gold로 첫 매대를 열고 판매 수익으로 성장 루프에 진입하게 한다. 매대 해금 비용 초안은 1번 매대 5 Gold, 2번 매대 250 Gold, 3번 매대 12.5k Gold로 둔다. 직원과 추가 손님은 스테이지 업그레이드 리스트를 통해 늘어난다.

다음 스테이지 이동 비용은 현재 스테이지에서 가장 비싼 상품의 최종 레벨업 비용보다 1.5배 높은 값으로 산출한다.

참고 게임에서는 스테이지 클리어 시 Common Chest x1 같은 보상이 관측된다. Malltopia도 스테이지 클리어 보상 데이터 자리는 준비하되, 상자 오픈과 보상 연출은 MVP 이후로 미룬다.

MVP 테마 목표는 `놀이동산 Theme / 8 Stages`다. 초기 검증 범위는 Stage 1~3으로 둔다. Stage 1은 상품 1개, Stage 2는 상품 2개, Stage 3은 상품 3개 구성으로 루프 확장성을 검증한다.

## 8. Product System

상품은 매장 수익의 기본 단위다.

각 상품은 다음 데이터를 가진다.

- 상품 ID
- 표시 이름
- 소속 스테이지
- 해금 비용
- 기본 판매가
- 생산 시간
- 기본 생산 수량
- 업그레이드 레벨
- 최대 레벨
- 가격 증가율
- 별 마일스톤
- 생산 슬롯 표시/활성화 조건

상품 매대 레벨을 올릴 때마다 판매 보상이 증가한다. 특정 레벨에 도달하면 별이 채워지고, 별 획득 시 해당 상품의 보상이 크게 증가한다. 기본 방향은 별 1개 획득마다 기존 보상 대비 2배 증가다.

상품 최대 레벨은 스테이지에 따라 달라질 수 있다.

예시:

- Stage 1: 최대 25레벨
- Stage 2: 최대 75레벨
- 후반 스테이지: 더 높은 최대 레벨 가능

### Product Star Milestone

별은 상품 매대 성장의 큰 단위다.

예시:

| Star | Required Level | Effect |
|---:|---:|---|
| 1 | 10 | 상품 보상 x2, Diamond +1 |
| 2 | 25 | 상품 보상 x2, Diamond +1, 추가 생산 슬롯 조건 가능 |
| 3 | 50 | 상품 보상 x2, Diamond +1 |
| 4 | 75 | 상품 보상 x2, Diamond +1, 추가 생산 슬롯 조건 가능 |
| 5 | 100 | 상품 보상 x2, Diamond +1 |
| 6 | 130 | 상품 보상 x2, Diamond +1 |
| 7 | 170 | 상품 보상 x2, Diamond +1 |
| 8 | 200 | 상품 보상 x2, Diamond +1 |
| 9 | 235 | 상품 보상 x2, Diamond +1 |
| 10 | 270 | 상품 보상 x2, Diamond +1 |
| 11 | 310 | 상품 보상 x2, Diamond +1 |
| 12 | 350 | 상품 보상 x2, Diamond +1 |

별은 최대 12개까지 지원한다. 모든 상품이 12별까지 가는 것은 아니며, 스테이지와 상품별 `maxStarCount`로 제한한다. MVP 기준은 Stage 1은 2별, Stage 2는 4별, Stage 3은 4별로 시작한다.

별이 증가하면 화면 상단에 `보너스 수익 2배` 같은 짧은 안내 토스트를 띄운다. Diamond는 별 1개가 채워질 때마다 1개씩 지급하고, 헤더의 Diamond 수량에 즉시 반영한다.

별 진행 그래프는 현재 별 구간 기준으로 표시한다. 예를 들어 Stage 1 솜사탕은 1~9레벨 동안 1번째 별 그래프가 차고, 10레벨에 별 1개가 활성화되면 그래프가 초기화된다. 이후 11~24레벨 동안 2번째 별 그래프가 차고, 25레벨에 별 2개가 활성화되며 최대 레벨 상태로 표시한다.

매대 레벨업을 길게 누르는 중에도 레벨, 판매 보상, 다음 레벨업 비용, 별 진행도는 매 레벨업마다 즉시 갱신한다. 별 달성, 수익 2배, Diamond 지급 같은 토스트는 업그레이드 흐름을 멈추지 않는다. 새 토스트가 발생하면 기존 2초 유지 토스트를 기다리지 않고 최신 토스트로 즉시 교체한다.

### Product Station Popup UX

상품 스테이션은 상태에 따라 다른 팝업을 표시한다.

| State | Popup | Main Contents |
|---|---|---|
| Before Unlock | Product Unlock Popup | 상품명, 해금 비용, 해금 버튼 |
| Unlocked | Product Station Info Popup | 현재 레벨, 별 개수, 다음 별 진행도, 판매 보상, 생산 시간, 레벨업 비용 |
| Level Up Success | Level Up Feedback | 증가한 레벨, 변경된 보상, 다음 별까지 진행도 |
| Star Reached | Star Bonus Toast | 보너스 수익 x2, Diamond +1 |

### Production Slot

생산 슬롯은 메인 캐릭터나 직원이 상품을 생산할 수 있는 작업 위치다.

- 기본 상품 해금 후 첫 생산 슬롯을 사용할 수 있다.
- 1번 매대는 추가 생산 라인을 제공하지 않는다.
- 2번 매대와 3번 매대는 1별, 즉 10레벨 달성 시 두 번째 생산 슬롯 박스가 나타난다.
- 추가 생산 슬롯 박스는 Gold를 요구하지 않는다. 플레이어가 한 번 터치하면 무료로 활성화된다.
- 플레이어가 생산 슬롯을 활성화하지 않고 기존 슬롯만으로 플레이하는 것도 가능하다.
- 생산 슬롯이 여러 개면 메인 캐릭터와 직원이 동시에 같은 상품을 생산할 수 있다.
- 스테이지마다 상품별 최대 생산 슬롯 수가 다를 수 있다.

## 9. Main Character Agent System

메인 캐릭터는 이동 조작 캐릭터가 아니라, 게임 루프를 수행하는 기본 자동 에이전트다.

기본 작업:

- 손님에게 이동한다.
- 손님 주문을 받는다.
- 생산 위치로 이동한다.
- 생산 시간 동안 원형 진행 게이지를 표시한다.
- 완성된 상품을 들고 주문한 손님에게 전달한다.
- 상품 1개 전달 완료 시마다 Gold 획득을 발생시킨다.

기본 능력치:

- 이동속도
- 생산속도
- 전달속도
- 판매 수익 보너스
- S급 상품 등장 확률

직원은 메인 캐릭터와 동일한 에이전트 구조를 사용하되, 현재 스테이지의 업그레이드 리스트에서 `직원 +1`을 구매했을 때 추가된다. 직원은 현재 스테이지에서만 활성화된다.

## 10. Character Roster System

캐릭터 로스터는 MVP 이후 Diamond로 구매할 수 있는 수집/편의 성장 요소다. 메인 캐릭터는 한 명만 선택할 수 있으며, 선택된 캐릭터의 낮은 수치 패시브가 자동 에이전트에 적용된다.

캐릭터는 장비처럼 강한 수익 배율을 담당하지 않는다. 장비가 핵심 성장축이라면, 캐릭터는 플레이 스타일, 외형, 약한 편의 보너스를 담당한다.

기본 방향:

| Rule | Decision |
|---|---|
| Acquisition | 상점에서 Diamond로 구매 |
| Equip Rule | 보유 캐릭터 중 1명 선택 |
| Power Level | 장비보다 낮은 수치의 패시브 |
| Main Stat Scope | 주인공 이동속도, 주인공 작업속도 중심 |
| Utility Scope | 장비 분해 보상 증가, 공용 스킬 효과 소폭 증가 |
| MVP | 실제 구매/교체는 구현하지 않고 데이터 자리만 준비 |

캐릭터 등급:

| Grade | Role | Example Passive Direction |
|---|---|---|
| Normal | 기본 캐릭터 | 보너스 없음 |
| Rare | 약한 이동 편의 | 주인공 이동속도 +5% |
| Unique | 약한 작업 편의 | 주인공 작업속도 +5% |
| Epic | 이동/작업 혼합 | 주인공 이동속도 +8%, 작업속도 +5% |
| Legendary | 낮은 유틸 보너스 | 장비 분해 Star Dust +1%, 공용 스킬 효과 +1% |
| Mythic | 이벤트/특수 캐릭터 | 장비 분해 Star Dust +2%, 공용 스킬 효과 +2% |

구매 가격 초안:

| Grade | Diamond Cost |
|---|---:|
| Normal | 0 |
| Rare | 500 |
| Unique | 1200 |
| Epic | 2600 |
| Legendary | 4000 |
| Mythic | TBD, 이벤트 또는 특수 상품 후보 |

캐릭터 효과는 서로 다른 성장 시스템을 압도하지 않아야 한다. 특히 캐릭터에는 큰 수익 배율, 더블 확률, S상품 확률을 기본으로 넣지 않는다. 이런 강한 효과는 장비, 상자, 이벤트 장비 쪽에 둔다.

## 11. Equipment System

장비는 메인 캐릭터의 핵심 장기 성장 시스템이다.

장비 등급:

| Grade | Description |
|---|---|
| Normal | 기본 능력치 소폭 증가 |
| Rare | 주요 능력치 증가 |
| Unique | 특정 작업 특화 옵션 포함 |
| Epic | 복수 옵션과 높은 성장률 |
| Legendary | 강한 고유 효과와 직원 보너스 가능 |
| Mythic | 이벤트/특수 제작 중심의 최상위 희귀 장비 |

`Mythic`은 현재 `Hidden`의 대체 후보명이다. `Hidden`은 등급명보다는 숨겨진 획득 방식처럼 들리므로, 최상위 등급명은 `Mythic` 또는 한국어 표시명 `신화`를 우선 후보로 둔다.

장비 옵션 카테고리:

| Option Category | Example |
|---|---|
| Main Character Stat | 주인공 이동속도 증가, 주인공 작업속도 증가 |
| Main Character Sale | 주인공 수익 증가, 주인공 더블 확률, 주인공 S상품 확률, 주인공 S상품 보상 증가, 주인공 즉시생산 확률 |
| Staff Stat | 직원 이동속도 증가, 직원 작업속도 증가 |
| Staff Sale | 직원 수익 증가, 직원 더블 확률, 직원 S상품 확률, 직원 S상품 보상 증가, 직원 즉시생산 확률 |
| Shared Worker Stat | 전체 작업자 이동속도 증가, 전체 작업자 작업속도 증가 |
| Customer Flow | 손님 이동속도 증가, 손님 주문속도 증가 |
| Income | 전체 판매 수익 증가, 특정 상품 판매 수익 증가 |
| Stage Specialty | 특정 스테이지 상품군 수익 증가 |

MVP에서는 장비 시스템을 완성 구현하지 않는다. 단, 이후 확장을 위해 메인 캐릭터 능력치와 장비 옵션이 들어갈 데이터 구조는 고려한다.

용어 기준:

- `주인공`은 메인 캐릭터만 의미한다.
- `직원` 또는 `전체 직원`은 해금/고용된 직원만 의미하며 주인공은 제외한다.
- 주인공과 직원을 모두 포함하는 옵션은 `전체 작업자`라고 부른다.

### Equipment Base Income

모든 장비는 기본 공통 능력으로 `기본 수익 증가`를 가진다.

- Normal과 Rare는 서브 옵션 없이 기본 수익 증가만 가진다.
- 장비 레벨업 또는 등급 상승 시 기본 수익 증가만 성장한다.
- 서브 옵션의 수치와 종류는 장비 레벨업으로 변하지 않는다.
- 기본 수익 증가는 확률이 아니라 항상 적용되는 배율이다.

권장 표기:

```text
baseIncomeMultiplier = x1.2, x2.0, x5.0...
```

### Equipment Sale Formula

수익 관련 효과는 중복 적용 가능하다. 기본 수익 증가, 더블, S상품이 모두 적용될 수 있다.

```text
finalSaleGold =
  levelBaseSaleGold
  * starMultiplier
  * productRewardUpgradeMultiplier
  * equipmentBaseIncomeMultiplier
  * incomeOptionMultiplier
  * characterPassiveIncomeMultiplier
  * activeShopBoostMultiplier
  * doubleMultiplier
  * sGradeMultiplier
```

예시:

```text
기본 상품 판매가 = 10 Gold
장비 기본 수익 = x2
더블 발동 = x2
S상품 발동 = x3

최종 판매가 = 10 * 2 * 2 * 3 = 120 Gold
```

더블과 S상품은 서로 배타적이지 않다. 두 효과가 동시에 발동하면 둘 다 곱한다.

### Equipment Slot Identity

장비 부위별 옵션 성격은 다르게 둔다. 같은 등급이라도 부위별 옵션 풀이 달라야 장비 조합 선택지가 생긴다.

| Slot | Main Identity | Example Options |
|---|---|---|
| Tool | 주인공 판매/작업 특화 | 주인공 수익 증가, 주인공 더블 확률, 주인공 S상품 확률, 주인공 즉시생산 확률 |
| Hat | 직원 운영 특화 | 직원 작업속도, 직원 이동속도, 직원 더블 확률, 직원 즉시생산 확률 |
| Top | 운영/수익 혼합 성장 | 전체 수익 증가, 전체 작업자 작업속도, 손님 주문속도, 주인공/직원 수익 증가 |
| Bottom | 이동/동선 효율 특화 | 주인공 이동속도, 직원 이동속도, 손님 이동속도, 전체 작업자 이동속도 |

### Equipment Grade Option Rule

| Grade | Base Income | Sub Option Rule |
|---|---|---|
| Normal | 낮음 | 옵션 없음 |
| Rare | Normal보다 높음 | 옵션 없음 |
| Unique | 높음 | 옵션 1개 |
| Epic | Unique보다 높음 | 더 좋은 옵션 1개 |
| Legendary | 매우 높음 | 옵션 2개 |
| Mythic | 최고 | 옵션 3개 |

등급이 높아질수록 기본 수익 증가가 커지고, 옵션 풀도 더 강한 효과를 포함한다.

### Equipment Special Options

즉시생산 옵션:

- 생산 작업을 시작할 때 확률로 발동한다.
- 발동하면 해당 생산의 생산 시간을 건너뛰고 즉시 상품을 만든다.
- 주인공, 직원, 전체 작업자 대상 옵션으로 나눌 수 있다.
- 너무 강한 옵션이므로 Epic 이상 옵션 풀부터 사용한다.

손님 이동속도 옵션:

- 손님이 입장 지점에서 주문 위치로 걸어오는 속도와 상품 수령 후 퇴장하는 속도에 적용한다.
- 매장이 커질수록 체감되는 동선 압축 옵션이다.

손님 주문속도 옵션:

- 손님이 주문 위치에 도착한 뒤 주문을 확정하기까지의 기본 대기 시간을 줄인다.
- 기본 주문 대기 시간은 1~2초 범위로 두고, 장비 옵션으로 줄일 수 있다.
- 최소 주문 대기 시간은 밸런스 단계에서 별도 제한값을 둔다.

### Equipment Growth

장비는 중복 습득이 가능하다.

장비 레벨:

- 모든 장비의 최대 레벨은 30이다.
- 장비 레벨업에는 같은 등급 또는 하위 등급의 장비를 재료로 사용할 수 있다.
- 레벨업 경험치와 등급별 재료 경험치 값은 밸런스 단계에서 확정한다.

장비 등급 상승:

- 기본 등급 상승은 30레벨 동일 부위, 동일 장비 아이템 2개를 합성해 상위 등급으로 올리는 방식이다.
- Normal -> Rare -> Unique -> Epic 구간은 이 기본 합성 규칙을 사용한다.
- Legendary부터는 완제품이 직접 드랍되는 구조가 아니라 제작 문서와 재료를 모아 제작하는 구조로 둔다.
- Legendary 제작에는 해당 Legendary 제작 문서 1개와 하위 등급 재료 아이템 5개가 필요하다.
- Mythic 장비는 이후 이벤트 상자, 이벤트 제작 문서, 특수 재료를 통해 획득할 수 있게 설계한다.

장비 분해:

- 장비는 분해할 수 있다.
- 분해 시 `Star Dust` 재화를 지급한다.
- `Star Dust`는 캐릭터 상태창에서 메인 캐릭터 레벨을 올리는 데 사용한다.

상자에서 나오는 Legendary 보상은 완제품 장비가 아니라 제작 문서다. 참고 이미지처럼 `로봇 머리 Legendary Blueprint`를 획득하고, 필요한 하위 재료를 모아 제작 버튼을 누르는 구조를 기본 방향으로 둔다.

## 12. S Grade Product System

S급 상품은 일반 상품보다 높은 보상을 주는 특수 상품이다.

기본 정의:

- 상품 생산 완료 시 일정 확률로 S급 상품이 된다.
- S급 상품은 판매 시 기본 판매가보다 높은 Gold를 지급한다.
- 장비 옵션, 직원 능력치 업그레이드, 스테이지 추가 해금 효과로 S급 상품 등장 확률과 보상 배율을 높일 수 있다.

MVP에서는 S급 상품을 구현하지 않는다. 장비 시스템과 함께 장기 성장 요소로 설계한다.

## 13. Customer System

MVP 손님은 일반 손님 1종이다.

행동 흐름:

1. 입장 지점에서 생성
2. 주문 위치로 이동
3. 메인 캐릭터 또는 직원에게 주문
4. 주문 상품 말풍선 표시
5. 상품 생산과 전달 대기
6. 상품 수령
7. 퇴장 지점으로 이동
8. 제거

MVP 기본 손님 흐름은 `one-in-one-out`이다. 손님 +1 업그레이드 전에는 동시에 1명만 존재한다. 손님은 주문 후 상품을 받을 때까지 매장 안의 주문 위치에서 대기하며, 이 대기는 별도 줄서기/대기열 시스템이 아니다.

첫 손님은 해당 스테이지의 첫 상품을 주문한다. 첫 상품은 아직 해금 전이어도 주문 대상으로 허용되며, 이 주문이 첫 매대 해금칸 점멸을 유도한다. 첫 상품 해금 이후의 주문 대상은 `매대 해금됨 && 생산 슬롯 활성화됨` 조건을 만족하는 상품으로 제한한다. 해금되지 않은 두 번째 이후 상품은 손님이 주문하지 않는다.

매대가 여러 개 해금되어 있으면 한 주문에 최대 2종 상품이 들어갈 수 있다. 주문 가능한 상품 풀은 현재 해금되고 생산 슬롯이 활성화된 상품만 포함한다. 상품 선택, 상품 종류 수, 상품별 주문 수량은 모두 균등 랜덤으로 처리한다. 상품이 추가로 해금되면 주문 가능한 상품 풀에 추가되고, 상품별 선택 확률은 다시 균등 분배된다. 상품별 주문 수량은 1~3개 중 균등 랜덤이며, 예를 들어 솜사탕 2개, 닭꼬치 3개 또는 솜사탕 1개와 닭꼬치 1개처럼 구성될 수 있다.

손님 +1 이후에는 카운터를 기준으로 손님 위치를 배치한다. 카운터가 일렬 4칸이면 우선 카운터 앞에 4명을 일렬로 세운다. 좌우 공간이 있으면 좌우로 확장하고, 빈 공간이 부족하면 뒤쪽 줄로 fallback한다.

손님 수 증가는 스테이지 업그레이드 리스트의 `손님 +1` 항목을 통해 이루어진다. 새로운 손님 타입이나 구매 성향은 장기 성장 시스템으로 남겨둔다.

## 14. Staff System

직원은 스테이지 업그레이드 리스트의 `직원 +1` 항목을 통해 현재 스테이지에 추가되는 자동 에이전트다. 직원은 메인 캐릭터와 같은 이동/작업 루프를 사용한다.

MVP에서는 메인 캐릭터와 직원 모두 주문, 생산, 전달을 처리할 수 있는 범용 에이전트로 시작한다. 이후 맵이 카운터/손님 응대 구역과 생산 구역으로 나뉘는 경우 서비스 직원과 생산 직원 역할을 분리한다.

직원 능력치 보너스 출처:

- 메인 캐릭터가 착용한 장비의 직원 보너스 옵션
- 하단 메뉴 업그레이드 리스트의 직원 능력치 증가
- 장기 성장 시스템의 전역 직원 능력치 증가

직원 보너스 적용 범위:

- 스테이지 업그레이드 리스트에서 구매한 직원 보너스는 해당 스테이지 직원에게만 적용한다.
- 전역 직원 보너스는 모든 스테이지 직원에게 적용할 수 있다.
- 장비 옵션은 전역 옵션과 특정 스테이지 옵션을 구분해서 설계한다.

### Future Role Split

MVP 이후 맵이 물리적으로 분리되면 직원 역할을 나눌 수 있다.

서비스 직원은 주문/전달 루프 일부를 대신 수행한다. 식당 스테이지라면 서빙 유닛, 쇼핑 스테이지라면 판매/서비스 담당처럼 스테이지 테마에 맞춰 이름과 연출이 바뀔 수 있다.

생산 직원은 생산 또는 상품 준비 루프를 보조한다.

식당 스테이지라면 요리사, 쇼핑 스테이지라면 생산/물류 직원처럼 표현할 수 있다. 해금된 직원은 해당 스테이지에서만 활성화된다.

MVP 범용 직원의 주요 능력:

- 이동 속도
- 작업 시간
- 업그레이드 레벨

MVP에서는 직원 동선을 단순화한다. 대기 지점, 주문 지점, 생산 지점, 전달 지점 사이를 waypoint 기반으로 이동한다.

주문 접수는 메인 캐릭터 또는 직원 1명이 손님 1명의 전체 주문을 받는다. 주문에 상품 2개가 있거나 2종 상품이 섞여 있어도 주문 접수 자체는 한 번에 끝난다.

주문 접수 후에는 주문 안의 상품을 `상품 1개 단위 포장 작업`으로 쪼갠다. 빈 메인 캐릭터 또는 직원은 포장 작업 1개를 가져가 생산/포장하고, 완성된 상품을 해당 주문 손님에게 전달한다. 기본적으로 주인공과 직원은 1회에 상품 1개만 처리한다. 이후 자원 업그레이드나 장비 옵션으로 `1회 포장 시 2개 획득 확률`을 추가할 수 있지만, 최대 처리 수량은 2개까지로 제한한다.

Gold는 주문 전체 완료 시점이 아니라 상품 1개가 손님에게 전달될 때마다 즉시 지급한다.

## 15. Upgrade System

업그레이드는 하단 푸터 영역의 상점 또는 메뉴 아이콘을 통해 접근한다. 아이콘을 누르면 현재 스테이지의 업그레이드 리스트 팝업이 열린다.

모든 행동 가능 액션은 작은 원형 안의 화살표 표시를 사용한다. 이 표시는 해금, 레벨업, 생산 슬롯 활성화, 스테이지 업그레이드, 다음 스테이지 이동, 전역 기능 해금처럼 유저가 지금 행동할 수 있는 곳에 붙는다.

표시 규칙:

| Action Type | Currency | Indicator Rule |
|---|---|---|
| Stage Gold Action | Gold | 구매 가능하면 계속 표시 |
| Product Level Up | Gold | 구매 가능하면 매대 또는 팝업에 계속 표시 |
| Production Slot Activation | Star condition | 조건 충족 후 무료 활성화 전까지 표시 |
| Next Stage Move | Gold | 조건 충족과 비용 보유 시 계속 표시 |
| New Feature Or Global Upgrade | Diamond or feature state | 새 기능/새 업그레이드가 등장하면 표시, 유저가 확인하면 숨김 |
| Equipment/Chest/Vault Notice | Diamond or item state | 새 항목 확인 전까지만 표시 |

Gold는 현재 스테이지 안에서 쓰는 단발성 진행 자원이다. 따라서 Gold로 가능한 스테이지 내부 업그레이드는 유저가 확인해도 사라지지 않고, 구매 가능 상태인 동안 계속 표시한다. 반대로 전역 기능, 장비, 금고, 상자처럼 유저 계정이나 기능 해금과 관련된 알림은 유저가 확인하면 숨기고, 새로운 업그레이드나 새 항목이 생길 때 다시 표시한다.

업그레이드 리스트 예시:

| Upgrade | Effect | Example Cost |
|---|---|---:|
| 솜사탕 생산속도 증가 | 생산속도 x2 | 50 |
| 솜사탕 판매가격 증가 | 판매가격 x2 | 1.8k |
| 손님 증가 | 손님 +1 | 1.2m |

Stage 1은 튜토리얼 성격의 작은 스테이지이므로 직원 +1 업그레이드를 넣지 않는다. 직원 +1은 Stage 2 이후부터 도입하는 후보로 둔다.

업그레이드 비용 표기는 `1, 10, 999, 1k, 1m, 1b, 1t, 1aa, 1ab, 1ac...` 형태의 축약 표기를 사용한다.

### Product Level Upgrade

효과:

- 판매가 증가
- 최대 레벨 달성을 통한 다음 스테이지 조건 충족

매대 레벨업 버튼은 길게 누르기를 지원한다. 유저가 버튼을 누르고 있는 동안 Gold가 충분하고 최대 레벨에 도달하지 않았다면 일정 간격으로 레벨업을 반복한다. 길게 누르는 시간이 길어질수록 반복 속도가 빨라지고, 최소 간격에 도달하면 1초에 수십 레벨까지 빠르게 올라갈 수 있다. 길게 누르기 반복 업그레이드는 매대 레벨업 전용으로 시작한다.

반복 업그레이드 중 별 달성, Diamond 지급, 수익 2배 같은 피드백이 발생해도 반복 업그레이드는 멈추지 않는다. 상품 정보 팝업의 레벨, 수익, 비용 표시는 매 업그레이드마다 즉시 갱신하고, 토스트는 최신 이벤트로 교체한다.

### Stage Upgrade

효과:

- 상품 생산속도 증가
- 상품별 보상 배율 증가
- 현재 스테이지 손님 수 증가
- 현재 스테이지 직원 수 증가
- 현재 스테이지 직원 이동속도 또는 생산속도 증가
- 기다리지 않고 상품 바로 추출
- S급 상품 생산 확률 증가

### Equipment Upgrade

효과:

- 이동속도 증가
- 작업속도 증가
- 판매 수익 증가
- S급 상품 관련 확률 또는 보상 증가
- 해당 스테이지 또는 전역 직원 능력치 보너스

## 16. Economy

### Currency

MVP 기본 재화는 Gold와 Diamond를 사용한다.

| Diamond | 별 마일스톤 보상으로 지급, 헤더 표시, 저장 | 상자 구매, 금고 또는 전역 업그레이드, 특수 능력치 증가 |
| Star Dust | MVP 이후 장비 분해로 획득 | 캐릭터 레벨업 또는 캐릭터 상태창 성장 |

### Income

- 상품 판매
- 별 마일스톤 달성 시 Diamond +1

### Spending

- 스테이지 업그레이드 구매
- 다음 스테이지 이동
- 장비 획득 또는 강화
- 상자 구매
- 시간제 수익 부스터 구매
- 캐릭터 구매

## 17. Shop Item System

상점 아이템은 MVP 이후 Diamond 또는 결제 상품으로 구매할 수 있는 장기 수익화/성장 요소다.

상점 아이템 종류:

| Type | Example |
|---|---|
| Chest | 장비, 제작 문서, 재료 획득 |
| Timed Income Boost | 15분 동안 수익 x2, 5분 동안 수익 x5 |
| Character | Rare 캐릭터 500 Diamond, Unique 캐릭터 1200 Diamond |

시간제 수익 부스터:

- 일정 시간 동안 상품 판매 Gold에 배율을 적용한다.
- 서로 다른 수익 부스터가 동시에 활성화되면 배율은 곱연산으로 중첩된다.
- 같은 수익 부스터를 다시 구매하면 배율을 중복하지 않고 남은 지속시간을 연장한다.
- 부스터는 장비 기본 수익, 장비 옵션, 더블, S상품과 함께 최종 판매 Gold 계산에 포함된다.

예시:

```text
기본 상품 판매가 = 10 Gold
15분 수익 x2 부스터 활성
5분 수익 x5 부스터 활성

최종 판매가 = 10 * 2 * 5 = 100 Gold
```

수익 부스터 최종 공식:

```text
finalSaleGold =
  levelBaseSaleGold
  * starMultiplier
  * productRewardUpgradeMultiplier
  * equipmentBaseIncomeMultiplier
  * incomeOptionMultiplier
  * characterPassiveIncomeMultiplier
  * activeShopBoostMultiplier
  * doubleMultiplier
  * sGradeMultiplier
```

## 18. Global Upgrade And Vault System

금고 또는 전역 업그레이드는 Diamond를 사용하는 장기 성장 시스템이다. 장비가 캐릭터/직원 능력치 중심이라면, 금고는 계정 단위의 편의와 경제 보너스를 담당한다.

예시:

| Upgrade | Effect Example |
|---|---|
| Ad Boost Duration | 광고 부스트 지속시간 5분 -> 6분 -> 7분 |
| Ad Boost Multiplier | 광고 수익 x2 -> x2.5 -> x3 |
| Offline Reward | 오프라인 보상 효율 증가 |
| Starting Cash | 새 스테이지 시작 보너스 Gold 증가 |
| Customer Flow | 손님 이동속도, 손님 주문속도, 방문 효율 증가 |

MVP에서는 실제 구매와 효과 적용을 구현하지 않는다. 다만 Diamond 저장과 데이터 확장 위치는 준비한다.

## 19. Offline Reward

앱 종료 후에도 매장은 운영된 것으로 간주한다.

MVP 계산 방식:

```text
effectiveOfflineSec = min(offlineSec, maxOfflineSec)
offlineGold = estimatedGoldPerSec * effectiveOfflineSec * offlineEfficiency
```

MVP 기본값:

- 최대 인정 시간: 3시간
- 오프라인 효율: 25%

## 20. Ads

MVP에서는 실제 광고 SDK를 붙이지 않는다.

단, 보상형 광고를 나중에 붙일 수 있도록 인터페이스와 버튼 위치는 준비한다.

MVP 광고 보상 더미:

- 수익 2배 버튼: 비활성 또는 테스트 보상
- 기본 광고 부스트 예시: 5분 동안 Gold 획득 x2
- 금고 업그레이드에 따라 지속시간과 배율이 증가할 수 있음
- Diamond 지급 광고 버튼: 1회 50 Diamond, 일 5회 후보, MVP 제외

광고 Diamond 보상은 캐릭터, 상자, 금고 같은 Diamond 사용처가 열리는 패치에서 함께 검토한다. 하루 최대 250 Diamond를 기준으로 잡으면 Rare 캐릭터 500 Diamond는 광고만으로 약 2일, Unique 캐릭터 1200 Diamond는 약 5일 목표가 된다.

## 21. Firebase And Version Management

Malltopia는 MVP 기준으로 서버 DB 없이 로컬 저장 기반으로 개발한다. 단, 앱 버전 관리와 운영 설정을 위해 Firebase를 사용한다.

초기 Firebase 사용 범위:

| Firebase Feature | Purpose | MVP Role |
|---|---|---|
| Remote Config | 최소 지원 버전, 최신 버전, 강제 업데이트, 점검 상태, 데이터 버전 관리 | 사용 후보 |
| Analytics | 기본 퍼널, 스테이지 진행, 이탈 구간 확인 | 선택 |
| Crashlytics | 크래시 수집 | 선택 |
| Firestore/Realtime Database | 랭킹, 클라우드 저장, 이벤트 상태 | MVP 제외 |
| Auth | 기기 변경 동기화, 랭킹 유저 식별 | MVP 제외 |
| Cloud Functions | 랭킹 제출 검증, 보상 지급 검증 | MVP 제외 |

Remote Config 주요 키 후보:

| Key | Example | Meaning |
|---|---|---|
| min_supported_version | 0.1.0 | 이 버전 미만이면 강제 업데이트 |
| latest_version | 0.1.2 | 권장 업데이트 표시 기준 |
| force_update_required | false | 강제 업데이트 여부 |
| maintenance_mode | false | 점검 모드 여부 |
| master_data_version | 1 | 로컬 마스터 데이터 버전 |
| balance_version | 1 | 밸런스 설정 버전 |
| event_config_version | 0 | 이벤트 설정 버전 |
| android_store_url | TBD | Android 업데이트 이동 URL |
| ios_store_url | TBD | iOS 업데이트 이동 URL |

Firebase는 MVP에서 유저 진행 데이터의 원본 저장소가 아니다. Gold, Diamond, 상품 레벨, 보유 캐릭터, 장비 인벤토리 같은 진행 데이터는 로컬 JSON을 기준으로 저장한다.

이벤트 클리어 타임 랭킹과 기기 변경 동기화는 Firebase Auth, Firestore 또는 별도 서버 검증이 필요한 확장 기능으로 분리한다.

## 23. MVP Success Criteria

MVP는 다음 조건을 만족하면 1차 완료로 본다.

- 손님이 자동으로 입장하고 구매 행동을 반복한다.
- 메인 캐릭터가 자동으로 손님 주문, 생산, 전달을 수행한다.
- 플레이어가 해금 가능한 칸을 클릭해 상품 매대와 생산 테이블을 열 수 있다.
- 상품 판매로 Gold가 증가한다.
- Gold로 상품을 업그레이드할 수 있다.
- 상품 별 마일스톤 달성 시 보상이 증가한다.
- 상품 별 마일스톤 달성 시 Diamond +1이 지급된다.
- 별 보너스 안내 토스트가 표시된다.
- 추가 생산 슬롯 활성화 후 같은 상품의 병렬 생산이 가능하다.
- Gold로 하단 메뉴 업그레이드를 구매할 수 있다.
- 손님 +1 업그레이드 후 스테이지 운영 규모가 커진다. 직원 +1은 Stage 2 이후 도입 후보로 검증한다.
- 상품 최대 레벨 달성 후 다음 스테이지 이동 버튼이 깜박인다.
- 앱 종료 후 저장값이 유지된다.
- 오프라인 보상이 계산된다.

## 24. Concept Questions To Resolve

다음 항목은 코드 구현 전에 추가 결정이 필요하다.

### Before MVP Code

- Stage 1~3의 11x20 고정 화면 좌표 배치를 Unity 화면에서 시각 검수하고 필요 시 미세 조정한다.
- Stage 1~3 상품 레벨 커브 확정 후 `가장 비싼 최종 레벨업 비용 x1.5` 기준의 다음 스테이지 이동 비용 산출값을 검수한다.
- Stage 1의 하단 업그레이드 3종 비용을 최종 확정한다.
- Stage 2 이후 직원 +1 업그레이드의 첫 등장 스테이지와 비용을 확정한다.
- 손님 +1 업그레이드 비용, 주문 확정 대기 시간, 손님 퇴장 후 다음 손님 등장 지연을 확정한다.
- 저장 파일 버전, 초기화 정책, 저장 슬롯 1개/다중 슬롯 여부를 확정한다.
- Unity에서 사용할 임시 아트 방식, 2D/3D 카메라 각도, 셀 크기 기준을 확정한다.

### After MVP Loop Validation

- 놀이동산 4~8스테이지의 상품 수, 배치, 최대 레벨을 확정한다.
- 3스테이지 검증 후 8스테이지 확장 시 플레이 시간이 얼마나 늘어나야 하는지 정한다.
- Diamond 사용처를 상자, 금고, 장비 강화 중 무엇부터 열지 정한다.
- 금고 업그레이드 중 광고 지속시간/광고 배율을 MVP 이후 몇 번째 패치에 넣을지 정한다.
- 직원 역할을 어느 테마/스테이지부터 서비스/생산으로 분리할지 정한다.
- 직원 보너스를 전역/스테이지 한정/역할 한정 중 어떤 단위로 나눌지 정한다.
- 캐릭터를 계정 단위 수집으로 둘지, 테마별 보너스를 가진 캐릭터로 나눌지 정한다.
- 스테이지 클리어 보상으로 상자를 실제 지급할지, 보상 연출만 먼저 둘지 정한다.
- 캐릭터 레벨업에 필요한 Star Dust 요구량과 레벨별 효과 증가폭을 정한다.
- 장비 획득처 중 일반 상자 외에 이벤트 상자, 스테이지 보상, 업적 보상을 어느 순서로 열지 정한다.
- Mythic 등급의 한국어 표시명을 `신화`로 확정할지, 다른 이름을 사용할지
- Legendary 제작 문서와 재료 아이템의 세부 레시피를 부위별로 고정할지, 장비별로 다르게 둘지
- S급 상품 발생 시점을 생산 시작, 생산 완료, 전달 완료 중 어디로 둘지
