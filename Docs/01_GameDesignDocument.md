# Malltopia Game Design Document

Version: 0.1  
Status: MVP planning  
Target: Android, iOS  

## 1. Concept

Malltopia는 작은 동네 구멍가게에서 시작해 편의점, 슈퍼마켓, 대형마트, 백화점, 메가몰로 성장하는 방치형 경영 시뮬레이션이다.

플레이어는 직접 이동하거나 상품을 판매하지 않는다. 플레이어의 역할은 매장 운영자이며, 상품 해금, 상품 업그레이드, 직원 고용, 직원 업그레이드, 매장 확장 판단을 통해 매장의 자동 운영 효율을 높인다.

## 2. Design Direction

MVP는 완전 자동 운영을 기본으로 한다.

- 손님은 자동으로 입장한다.
- 손님은 자동으로 상품을 선택한다.
- 손님은 자동으로 계산대로 이동한다.
- 기본 계산대와 기본 재입고는 자동으로 작동한다.
- 직원은 고용 후 계산 또는 진열 효율을 높인다.
- 플레이어는 업그레이드와 고용 결정을 통해 수익 속도를 높인다.

초반 탭 부스트, 수동 재입고, 수동 계산은 MVP에 포함하지 않는다.

### MVP Implementation Decisions

다음 결정은 MVP 코드 구현 기준으로 고정한다.

| Topic | Decision |
|---|---|
| Player input | 탭 부스트, 수동 재입고, 수동 계산 없음 |
| Checkout before cashier | 계산대가 기본 처리 시간으로 자동 결제 |
| Checkout after cashier | 계산원 고용 시 결제 처리 시간 감소 |
| Restock before restocker | 진열대가 상품별 기본 재입고 시간으로 자동 보충 |
| Restock after restocker | 진열 직원 고용 시 부족 재고를 더 빠르게 보충 |
| Customer movement | MVP는 waypoint 기반 이동 |
| Product choice | 해금되어 있고 재고가 있는 상품 중 랜덤 선택 |
| Save method | 로컬 JSON 저장 |

## 3. Genre

Idle Tycoon / Management Simulation

핵심 경험은 "작은 매장이 점점 바쁘고 효율적인 쇼핑 공간으로 변하는 것을 보는 만족감"이다.

## 4. Core Fantasy

플레이어는 작은 구멍가게의 점주로 시작한다. 처음에는 상품도 적고 손님도 적지만, 수익을 다시 투자하면서 더 많은 상품, 더 빠른 직원, 더 큰 매장을 갖춘 쇼핑 제국을 만든다.

## 5. Core Loop

1. 손님 입장
2. 상품 탐색
3. 상품 선택
4. 계산대 이동
5. 결제
6. Gold 획득
7. 상품 업그레이드
8. 직원 고용 또는 업그레이드
9. 매장 확장
10. 더 많은 손님과 더 높은 수익

## 6. Progression

| Stage | Name | Theme | Example Products |
|---|---|---|---|
| 1 | 동네 구멍가게 | 작은 생활 매장 | 과자, 음료, 라면 |
| 2 | 편의점 | 빠른 회전율 | 도시락, 커피, 우유, 아이스크림 |
| 3 | 슈퍼마켓 | 신선식품 | 채소, 과일, 정육, 생선 |
| 4 | 대형마트 | 대량 판매 | 생활용품, 전자제품, 주방용품 |
| 5 | 백화점 | 고가 상품 | 의류, 화장품, 명품 |
| 6 | 메가몰 | 초대형 소비 공간 | 자동차, 가구, 프리미엄 브랜드 |

MVP에서는 Stage 1만 구현한다.

## 7. MVP Scope

### Included

- Stage 1 동네 구멍가게
- 상품 3종: 과자, 음료, 라면
- 손님 1종: 일반 손님
- 직원 2종: 계산원, 진열 직원
- 상품 해금
- 상품 업그레이드
- 직원 고용
- 직원 업그레이드
- 매장 업그레이드
- Gold 수익
- 로컬 저장
- 오프라인 보상
- 광고 보상 구조의 더미 인터페이스

### Excluded

- VIP 손님
- 소비왕
- 단체 고객
- 상자 시스템
- 쇼핑몰 카드 시스템
- 이벤트 시스템
- 결제 상품
- 실제 광고 SDK 연동
- Stage 2 이상

## 8. Product System

상품은 매장 수익의 기본 단위다.

각 상품은 다음 데이터를 가진다.

- 상품 ID
- 표시 이름
- 소속 스테이지
- 해금 비용
- 기본 판매가
- 재입고 시간
- 진열 가능 재고량
- 업그레이드 레벨
- 최대 레벨
- 가격 증가율

상품 업그레이드는 판매가를 증가시킨다. MVP에서는 재입고 시간 감소나 재고량 증가를 상품 업그레이드 효과에 포함하지 않는다.

## 9. Customer System

MVP 손님은 일반 손님 1종이다.

행동 흐름:

1. 입장 지점에서 생성
2. 구매 가능한 상품 진열대 중 하나 선택
3. 진열대로 이동
4. 상품 획득
5. 계산대로 이동
6. 결제 완료 대기
7. 퇴장 지점으로 이동
8. 제거

상품 재고가 없으면 손님은 다른 상품을 선택한다. 구매 가능한 상품이 없으면 짧게 대기한 뒤 다시 탐색한다.

## 10. Staff System

### Cashier

계산원은 계산대에서 손님의 결제를 처리한다.

계산원이 없어도 계산대는 기본 속도로 결제를 처리한다. 계산원은 자동 운영을 해금하는 필수 조건이 아니라 결제 속도를 높이는 성장 요소다.

주요 능력:

- 계산 처리 시간
- 업그레이드 레벨

### Restocker

진열 직원은 재고가 부족한 상품 진열대를 보충한다.

진열 직원이 없어도 진열대는 상품별 기본 재입고 시간에 따라 자동으로 보충된다. 진열 직원은 재고 부족 시간을 줄이는 성장 요소다.

주요 능력:

- 이동 속도
- 진열 작업 시간
- 업그레이드 레벨

MVP에서는 직원 동선을 단순화한다. 창고 또는 대기 지점에서 진열대로 이동하고, 작업 완료 후 다시 대기 상태로 돌아간다.

## 11. Upgrade System

### Product Upgrade

효과:

- 판매가 증가

### Staff Upgrade

효과:

- 계산원: 계산 시간 감소
- 진열 직원: 진열 작업 시간 감소

### Store Upgrade

효과:

- 최대 동시 손님 수 증가
- 손님 입장 주기 감소

## 12. Economy

### Currency

MVP 기본 재화는 Gold 하나만 사용한다.

Diamond는 장기 기능으로 남겨둔다. MVP 데이터에는 Diamond 필드를 두지 않아도 된다.

### Income

- 상품 판매

### Spending

- 상품 해금
- 상품 업그레이드
- 직원 고용
- 직원 업그레이드
- 매장 업그레이드

## 13. Offline Reward

앱 종료 후에도 매장은 운영된 것으로 간주한다.

MVP 계산 방식:

```text
effectiveOfflineSec = min(offlineSec, maxOfflineSec)
offlineGold = estimatedGoldPerSec * effectiveOfflineSec * offlineEfficiency
```

MVP 기본값:

- 최대 인정 시간: 3시간
- 오프라인 효율: 25%

## 14. Ads

MVP에서는 실제 광고 SDK를 붙이지 않는다.

단, 보상형 광고를 나중에 붙일 수 있도록 인터페이스와 버튼 위치는 준비한다.

MVP 광고 보상 더미:

- 30분 수익 2배 버튼: 비활성 또는 테스트 보상
- Diamond 지급 버튼: 제외

## 15. Visual Direction

추천 방향:

- 2.5D 또는 탑다운 저폴리
- 작은 매장의 동선이 한눈에 보이는 카메라
- 진열대, 계산대, 입구, 출구가 명확히 구분되는 레이아웃
- 업그레이드될수록 손님과 직원 움직임이 많아지는 시각적 성장

## 16. MVP Success Criteria

MVP는 다음 조건을 만족하면 1차 완료로 본다.

- 손님이 자동으로 입장하고 구매 행동을 반복한다.
- 상품 판매로 Gold가 증가한다.
- Gold로 상품을 업그레이드할 수 있다.
- Gold로 직원 고용과 업그레이드를 할 수 있다.
- 기본 계산대와 기본 재입고가 자동 수행된다.
- 직원 고용 후 계산 또는 진열 효율이 증가한다.
- 매장 업그레이드로 손님 회전율이 증가한다.
- 앱 종료 후 저장값이 유지된다.
- 오프라인 보상이 계산된다.
