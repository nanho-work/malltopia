# Malltopia MVP Specification

Version: 0.1  
Status: Implementation reference  

## 1. Goal

MVP의 목표는 하나의 테마 안 3개 스테이지에서 다음 루프가 안정적으로 반복되는 상태를 만드는 것이다. 테마 전체 목표는 8개 스테이지지만, 먼저 3개 스테이지로 핵심 루프와 밸런스를 검증한 뒤 4~8스테이지를 확장한다.

```text
손님 입장 -> 메인 캐릭터 주문 접수 -> 상품 해금/생산 테이블 활성화 -> 생산 -> 전달 -> Gold 획득 -> 하단 메뉴 업그레이드 -> 다음 스테이지 준비
```

MVP는 재미 검증용 수직 슬라이스다. 전체 게임의 모든 시스템을 구현하지 않는다.

## 2. Scene

MVP는 단일 메인 씬으로 구성한다.

권장 씬 이름:

- `MainStage`

씬 구성 요소:

- 입구 지점
- 출구 지점
- 솜사탕 상품 매대 1개
- 솜사탕 생산 테이블 1개
- 메인 캐릭터 시작 지점
- 상품 생산 지점
- 직원 대기 지점
- 손님 이동 영역
- UI Canvas
- Footer Menu Button
- Stage Upgrade Popup
- Next Stage Button
- GameManager
- SaveManager

## 3. Stage Layout

MVP 검증 범위:

| Stage | Product Count | Products | Max Level | Max Stars | Purpose |
|---:|---:|---|---:|---:|---|
| 1 | 1 | 솜사탕 | 25 | 2 | 첫 주문, 첫 매대, 별 2개, 최대 레벨 검증 |
| 2 | 2 | 구슬아이스크림, 닭꼬치 | 75 | 4 | 복수 매대, 1별 생산 슬롯, 상품 선택 검증 |
| 3 | 3 | 팝콘, 핫도그, 츄러스 | 75 | 4 | 복수 생산 슬롯과 직원 효율 검증 |

기본 클리어 조건:

- 현재 스테이지의 모든 상품 매대를 해금해야 한다.
- 현재 스테이지의 모든 상품 매대를 최대 레벨까지 업그레이드해야 한다.
- 조건 충족 후 다음 스테이지 이동 버튼이 점멸한다.
- 이동 버튼을 눌렀을 때 필요 Gold를 지불하면 다음 스테이지로 이동한다.
- 다음 스테이지 이동 비용은 현재 스테이지 상품 중 가장 비싼 최종 레벨업 비용의 1.5배로 산출한다.

MVP Stage 1~3은 스크롤 없는 화면 고정형 맵이다. 폰 세로 화면을 11x20 그리드로 보고, x축 중앙은 `5`로 둔다.

좌표 기준:

| Axis | Range | Meaning |
|---|---|---|
| x | 0~10 | 화면 가로. x=5가 중앙 |
| y | 0~19 | 화면 세로. y=0이 상단 입구 방향 |

공통 배치 규칙:

- y=0: 입구/출구
- y=1~3: 손님 이동길과 fallback 대기 위치
- y=4: 카운터 앞 손님 주문 위치
- y=5: 카운터
- y=6~8: 매대, 생산 슬롯, 작업자 이동 공간
- y=9~12: 주인공/직원 이동 여백
- y=13~19: 장식/여백 영역

## 4. MVP Entities

### Main Character Agent

플레이어의 이동 조작 대상이 아닌 기본 자동 에이전트다. 메인 캐릭터는 주문 접수, 생산, 전달 루프를 수행한다.

상태:

- Idle
- MovingToCustomer
- TakingOrder
- WaitingForUnlock
- MovingToProduction
- Producing
- CarryingProduct
- DeliveringProduct
- CollectingGold

MVP 기준 기본 능력치:

- 매대 1칸 이동 시간
- 작업속도 배율
- 전달 처리 시간
- 판매 수익 보너스
- S급 상품 확률

상품 생산 시간은 캐릭터가 아니라 상품 데이터의 `baseProductionTimeSec`를 기준으로 한다. MVP에서는 스테이지 내 상품 순번에 따라 1번 상품 5초, 2번 상품 9초, 3번 상품 13초를 사용한다.

### Customer

일반 손님 1종만 구현한다.

상태:

- Spawned
- WaitingForOrder
- Ordering
- WaitingForProduct
- ReceivingProduct
- Leaving

`WaitingForProduct`는 손님이 상품을 받을 때까지 매장 안의 주문 위치에 머무르는 상태다. 별도 손님 대기열이나 줄서기 시스템을 의미하지 않는다.

### Product Stand

상품 매대는 상품의 시각적 표시와 판매 가능 상태를 담당한다.

상태:

- Locked
- UnlockAvailable
- Unlocked
- LevelUpAvailable
- StarMilestoneReached

### Production Table

생산 테이블은 해당 상품을 실제로 생산할 수 있게 하는 오브젝트다. 상품 매대가 해금된 뒤 별도 활성화칸을 통해 열린다.

실제 작업 가능 지점은 `Production Slot`으로 관리한다. 하나의 상품 매대는 여러 개의 생산 슬롯을 가질 수 있으며, 각 슬롯은 메인 캐릭터나 직원 한 명이 점유해 생산을 진행한다.

상태:

- Locked
- UnlockAvailable
- Unlocked
- Producing

### Product Star Milestone

상품 레벨이 특정 기준에 도달하면 별이 채워진다.

MVP 기본 규칙:

- 매대 레벨을 올릴 때마다 판매 보상이 증가한다.
- 별 마일스톤 도달 시 상품 보상이 기존 대비 2배 증가한다.
- 별 조건을 만족하면 추가 생산 슬롯 활성화칸이 표시될 수 있다.
- 추가 생산 슬롯은 표시 후 유저가 한 번 터치하면 무료로 활성화된다.
- 유저는 표시된 슬롯을 활성화하지 않은 채로도 플레이를 계속할 수 있다.

### Staff Agent

직원은 하단 메뉴의 `직원 +1` 업그레이드를 통해 현재 스테이지에 추가되는 자동 에이전트다.

MVP에서는 범용 직원 1종으로 시작한다. 범용 직원은 메인 캐릭터와 같은 주문 접수, 생산, 전달 루프를 수행할 수 있다.

이후 맵 구조가 카운터/손님 응대 구역과 생산 구역으로 분리되면 직원 역할을 주문/전달 직원, 생산 직원으로 나눌 수 있다. 직원 보너스는 현재 스테이지에 적용되는 업그레이드 효과의 영향을 받는다.

## 5. Functional Requirements

| ID | Requirement | MVP Rule |
|---|---|---|
| FR-001 | 손님 자동 생성 | 기본 maxCustomers=1. 손님이 상품을 받고 퇴장한 뒤 다음 손님 생성. 손님 +1 이후에는 현재 손님 수가 최대치보다 낮을 때 생성 |
| FR-002 | 주문 접수 | 메인 캐릭터가 손님에게 이동해 주문을 받음 |
| FR-003 | 주문 말풍선 | 손님 머리 위에 주문 상품 아이콘 표시 |
| FR-004 | 주문 가능 상품 제한 | 첫 손님은 첫 상품 주문. 이후에는 해금되고 생산 슬롯이 활성화된 상품만 주문 |
| FR-005 | 복수 상품 주문 | 해금된 매대가 여러 개면 한 주문에 최대 2종 상품, 상품별 1~3개 수량 균등 랜덤 |
| FR-006 | 해금 가능 표시 | 첫 상품 주문 후 주문 상품 해금칸이 깜박임 |
| FR-007 | 상품 해금 정보창 | 해금칸 클릭 시 상품명과 해금 필요 Gold 표시 |
| FR-008 | 매대 해금 | Gold를 지불하면 레벨 1 상품 매대가 열림 |
| FR-009 | 생산 테이블 활성화 | 매대 해금 후 생산 테이블 활성화칸 표시, 터치 시 무료 활성화 |
| FR-010 | 생산 | 생산 테이블 활성화 후 메인 캐릭터가 생산 위치로 이동해 생산 |
| FR-011 | 생산 진행 표시 | 생산 중 원형 게이지 표시 |
| FR-012 | 전달 | 생산 완료 후 메인 캐릭터 또는 직원이 상품을 주문한 손님에게 전달 |
| FR-013 | Gold 획득 | 상품 1개 전달 완료 시마다 상품 판매가만큼 Gold 증가 |
| FR-014 | Footer Menu | 하단 메뉴 아이콘 클릭 시 업그레이드 리스트 팝업 표시 |
| FR-015 | 상품 레벨 업그레이드 | Gold를 지불하고 상품 레벨 증가 |
| FR-016 | 별 마일스톤 | 특정 레벨 도달 시 별 증가와 보상 x2 적용 |
| FR-017 | 별 보상 지급 | 별이 증가할 때 Diamond +1 지급 |
| FR-018 | 별 보너스 안내 | 별 달성 시 `보너스 수익 2배` 안내 토스트 표시 |
| FR-019 | 생산 슬롯 조건 표시 | 별 조건 충족 시 추가 생산 슬롯 활성화칸 표시 |
| FR-020 | 생산 슬롯 활성화 | 표시된 추가 생산 슬롯 활성화칸을 한 번 터치해 무료 활성화 |
| FR-021 | 생산속도 업그레이드 | Gold를 지불하고 상품 생산속도 증가 |
| FR-022 | 보상 업그레이드 | Gold를 지불하고 상품 보상 배율 증가 |
| FR-023 | 손님 증가 | Gold를 지불하고 현재 스테이지 최대 손님 수 +1 |
| FR-024 | 직원 증가 | Gold를 지불하고 현재 스테이지 직원 수 +1 |
| FR-025 | 다음 스테이지 버튼 | 필수 조건 충족 시 하단 다음 스테이지 버튼 점멸 |
| FR-026 | 다음 스테이지 비용 | 현재 스테이지에서 가장 비싼 최종 레벨업 비용 x1.5를 표시하고 지불 |
| FR-027 | 저장 | Gold, Diamond, 해금 상태, 레벨, 생산 슬롯, 스테이지 업그레이드 상태 저장 |
| FR-026 | 오프라인 보상 | 마지막 저장 시간 기준으로 보상 계산 |
| FR-027 | 구매 가능 표시 | Gold로 가능한 스테이지 내부 액션은 구매 가능 동안 원형 화살표 표시 유지 |
| FR-028 | 신규 기능 표시 | Diamond/전역/기능성 신규 업그레이드는 유저가 확인하면 표시 숨김 |
| FR-029 | 길게 눌러 매대 업그레이드 | 매대 레벨업 버튼을 누르고 있는 동안 반복 레벨업 |

## 6. UI Requirements

### Main HUD

필수 표시:

- 현재 Gold
- 현재 Diamond
- 현재 Theme 이름
- 현재 Stage 번호
- 진행 중인 주문 또는 생산 상태
- 설정 버튼

### Footer Menu

필수 표시:

- 업그레이드 리스트 버튼
- 구매 가능한 Gold 업그레이드가 있을 때 원형 화살표 표시
- 다음 스테이지 이동 버튼
- 다음 스테이지 이동 가능 시 버튼 점멸

### Action Available Indicator

작은 원형 안의 화살표 표시를 공통 사용한다.

표시 대상:

- 매대 해금칸
- 생산 테이블 활성화칸
- 생산 슬롯 활성화칸
- 상품 레벨업 가능 상태
- 하단 업그레이드 리스트 버튼
- 스테이지 업그레이드 항목
- 다음 스테이지 이동 버튼
- MVP 이후 장비, 금고, 상자, 신규 기능 알림

표시 규칙:

- Gold로 진행하는 스테이지 내부 액션은 구매 가능하면 계속 표시한다.
- 유저가 확인만 해도 사라지지 않는다.
- Diamond, 장비, 금고, 상자, 신규 기능처럼 전역 또는 기능성 알림은 유저가 해당 화면을 확인하면 숨긴다.
- 숨긴 알림은 새 항목, 새 레벨, 새 기능이 등장했을 때 다시 표시한다.

### Product Unlock Popup

매대 오픈 전 표시:

- 상품 이름
- 해금 필요 Gold
- 해금 버튼
- 닫기 버튼

### Product Station Info Popup

각 상품별 표시:

- 상품 이름
- 현재 레벨
- 별 개수
- 다음 별까지의 진행도
- 현재 판매가
- 생산 시간
- 업그레이드 비용
- 업그레이드 버튼
- 업그레이드 가능 표시
- 길게 눌러 반복 업그레이드 입력
- 잠금 상태일 경우 해금 비용

### Product Feedback Toast

상품 레벨업 또는 별 마일스톤 달성 시 짧게 표시:

- 레벨업 성공
- 보너스 수익 2배
- Diamond +1

표시 규칙:

- 안내 토스트는 약 2초 후 자동으로 닫힌다.
- 새 안내 토스트가 발생하면 기존 토스트를 기다리지 않고 즉시 교체한다.
- 매대 레벨업을 길게 누르는 중에도 레벨, 수익, 업그레이드 비용, 별 진행도 표시는 매 업그레이드마다 즉시 갱신한다.
- 별 달성, Diamond 지급, 보너스 수익 토스트는 반복 업그레이드를 멈추지 않는다.
- 유저가 다른 화면이나 다른 오브젝트를 터치하면 즉시 닫힌다.
- 매대 정보 팝업, 해금 팝업 같은 조작 팝업도 외부 영역 터치 시 닫힌다.

- 현재 Theme ID

- 생산 슬롯 활성화 여부
- 상품 해금 여부
- 상품 레벨
- 활성화된 생산 슬롯 ID 목록
- 스테이지 업그레이드 구매 상태

- 다음 스테이지 이동 조건 충족 여부
- 마지막 저장 시각
- 누적 플레이 시간

- 로컬 JSON 파일

- Firebase는 MVP 유저 진행 데이터의 원본 저장소로 사용하지 않음
- 출시 전에는 변조 방지 또는 서버 검증 검토

### Firebase Version Management

MVP에서 Firebase를 사용하는 경우 우선 범위는 Remote Config 기반 버전 관리다.

관리 항목:

- 최소 지원 버전
- 최신 권장 버전
- 강제 업데이트 여부
- 점검 모드
- 마스터 데이터 버전
- 밸런스 버전
- 이벤트 설정 버전

이벤트 랭킹, 기기 변경 동기화, 결제 검증, 보상 검증은 Firebase Remote Config만으로 해결하지 않는다. 해당 기능은 Firebase Auth, Firestore, Cloud Functions 또는 별도 서버 설계가 필요한 MVP 이후 기능으로 분리한다.

## 8. Offline Reward Requirements

조건:

- 앱 시작 시 마지막 저장 시각이 존재해야 한다.
- 오프라인 시간이 60초 이상일 때만 팝업을 표시한다.
- 최대 인정 시간은 3시간이다.

계산:

```text
effectiveOfflineSec = min(offlineSec, 10800)
reward = estimatedGoldPerSec * effectiveOfflineSec * 0.25
```

`estimatedGoldPerSec`은 MVP에서는 현재 상품 판매가, 손님 퇴장 후 다음 손님 등장 지연, 주문/전달 처리 속도, 해금 상품 수를 기반으로 단순 추정한다.

## 10. Acceptance Criteria

MVP는 다음 항목을 모두 만족해야 완료로 본다.

- 메인 캐릭터가 자동으로 손님에게 이동해 주문을 받을 수 있다.
- 손님 주문 말풍선이 표시된다.
- 해금 가능한 상품칸이 깜박이고 클릭 시 정보창이 표시된다.
- Gold를 지불해 상품 매대를 해금하고, 생산 테이블은 표시된 칸 터치로 활성화할 수 있다.
- 상품 레벨이 별 마일스톤에 도달하면 별이 채워지고 보상이 증가한다.
- 상품 별 마일스톤 달성 시 Diamond가 1 증가한다.
- 별 달성 후 보너스 안내 토스트가 표시된다.
- 구매 가능한 Gold 액션에는 원형 화살표 표시가 계속 노출된다.
- 상품 레벨업 버튼을 누르고 있으면 Gold가 허용하는 동안 반복 업그레이드된다.
- 생산 중 원형 진행 게이지가 표시된다.
- 새 게임 시작 후 손님 1명과 메인 캐릭터 1명으로 솜사탕 판매 루프가 돌아간다.
- 하단 메뉴에서 스테이지 업그레이드 리스트를 열 수 있다.
- 솜사탕 생산속도 증가 업그레이드를 구매할 수 있다.
- 솜사탕 판매가격 증가 업그레이드를 구매할 수 있다.
- 손님 +1 업그레이드를 구매하면 현재 스테이지 최대 손님 수가 증가한다.
- 현재 스테이지의 모든 상품 최대 레벨 달성 후 다음 스테이지 버튼이 점멸한다.
- 다음 스테이지 이동 버튼 클릭 시 필요 Gold가 표시된다.
- 생산 완료 상품을 손님에게 전달하면 Gold가 증가한다.
- 앱을 종료하고 다시 실행해도 진행 상태가 유지된다.
- 오프라인 보상이 계산되어 지급된다.
- 5분 이상 플레이했을 때 명확한 성장감이 있다.

## 11. Implementation Defaults

아래 항목은 MVP 코드 구현 기준으로 사용한다.

- 손님 이동은 waypoint 기반으로 시작한다.
- 메인 캐릭터와 직원 이동도 waypoint 기반 자동 이동으로 시작한다.
- 첫 손님은 첫 상품을 주문한다. 이후 주문 상품은 해금되고 생산 슬롯이 활성화된 상품만 대상으로 한다.
- 한 주문은 최대 2종 상품을 포함할 수 있고, 상품 종류 수, 상품 선택, 상품별 수량 1~3개는 모두 균등 랜덤으로 결정한다.
- 주문 접수는 작업자 1명이 손님의 전체 주문을 받는다. 이후 포장/생산은 상품 1개 단위 작업으로 쪼개며, 빈 작업자가 하나씩 처리한다.
- 주인공과 직원은 기본적으로 1회에 상품 1개만 포장/운반한다. 향후 자원 업그레이드나 장비 옵션으로 최대 2개까지 확장할 수 있다.
- Gold는 주문 전체 완료가 아니라 상품 1개 전달마다 즉시 지급한다.
- 각 스테이지는 손님 1명, 메인 캐릭터 1명, 직원 0명으로 시작한다.
- 손님 +1 전에는 손님 1명이 입장, 주문, 상품 수령, 퇴장을 완료한 뒤 다음 손님이 입장한다.
- 각 스테이지는 시작 Gold 5로 시작한다.
- 각 스테이지의 첫 매대 해금 비용은 5 Gold로 시작한다.
- MVP 매대 해금 비용 초안은 1번 매대 5 Gold, 2번 매대 250 Gold, 3번 매대 12.5k Gold를 사용한다.
- 직원 수와 손님 수는 하단 메뉴 업그레이드 리스트에서 증가한다.
- 직원 보너스는 전역, 현재 스테이지 한정, 역할 한정으로 구분할 수 있게 설계한다.
- MVP Stage 1 상품 최대 레벨은 25, 최대 별은 2개로 시작한다.
- MVP Stage 2 상품 최대 레벨은 75, 최대 별은 4개로 시작한다.
- MVP Stage 3 상품 최대 레벨은 75, 최대 별은 4개로 시작한다.
- MVP Stage 1 생산칸은 1개만 사용한다.
- 1번 매대는 추가 생산 라인이 없다.
- 2번 매대와 3번 매대는 1별, 즉 10레벨 달성 시 두 번째 생산 슬롯이 표시되고, 유저 터치로 무료 활성화된다.
- 생산 슬롯 활성화 여부는 스테이지 클리어 조건에 포함하지 않는다.
- 매대 레벨업은 단일 터치와 길게 누르기 반복 입력을 모두 지원한다.
- 반복 레벨업은 매대 레벨업에만 적용한다.
- Diamond는 MVP에서 획득, 표시, 저장까지만 구현하고 사용처는 이후 패치로 미룬다.
- 저장은 로컬 JSON 파일로 구현한다.

## 12. Deferred Decisions

아래 항목은 MVP 구현 후 플레이 테스트 단계에서 결정한다.

- 놀이동산 4~8스테이지 상품 수와 배치

- NavMesh 전환 여부
